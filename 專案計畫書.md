## 1. 專案背景與目標

### 1.1 背景
社群在觀賞電競／體育賽事時，常以「猜勝隊」的方式增加參與感。原本以人工或試算表統計預測與積分，費時且易出錯。本專案以 Discord 機器人自動化整個流程：登錄比賽 → 玩家預測 → 結算計分 → 排行榜。

專案最初以 Replit + 單機 JSON 檔開發（`replit` 分支），驗證可行後，為了**資料可靠性與併發正確性**，遷移至 MongoDB 並改部署於 Zeabur（`main` 分支）。

### 1.2 核心目標
1. **正確**：在多人同時操作下，預測、計分、結算不得出現重複加分、覆蓋遺失等錯誤。
2. **易用**：全程斜線指令 + 自動完成，使用者零學習成本。
3. **可維運**：單檔結構、雲端託管、資料持久化，重啟不丟資料。

### 1.3 非目標（本階段不做）
- 跨伺服器共用排行榜（目前以 guild 為單位隔離）。
- 賽事資料外部 API 串接（比賽由管理員手動登錄）。
- 網頁管理後台。

## 2. 範圍與功能規格

### 2.1 角色
- **一般玩家**：預測、查詢自己的紀錄與排行榜、跟單。
- **管理員（Administrator）**：登錄比賽、結算、回溯、手動調分、刪除、重置。

### 2.2 功能清單
| 模組 | 功能 | 對應指令 |
|------|------|----------|
| 預測 | 預測勝隊、跟單、代客預測 | `/predict`、`/follow`、`/forcepredict` |
| 賽事管理 | 登錄、刪除、查詢 | `/register`、`/delete`、`/matches` |
| 結算 | 登錄結果、回溯 | `/result`、`/undoresult` |
| 計分 | 查分、排行、手動調分 | `/myscore`、`/leaderboard`、`/addscore`、`/subscore` |
| 統計 | 個人紀錄、票數統計 | `/mypredictions`、`/history` |
| 系統 | 說明、重置、彩蛋 | `/help`、`/resetall`、`/食物輪盤` |

### 2.3 業務規則
- 預測僅限「已登錄且未到開賽時間」的比賽。
- 同一玩家對同一比賽只能有一筆預測（後者覆蓋前者）。
- 每場比賽可設定不同分數（`match_points`），預設 1 分。
- 結算後不可重複結算；需修正時先 `/undoresult` 再重新 `/result`。
- 跟單只複製「未截止」比賽，且不覆蓋自己已有的預測。

## 3. 系統架構

### 3.1 整體
單一 Node.js 行程，內含 Discord Bot Client 與 Express 保活伺服器；資料持久化於 MongoDB。詳見 [README.md](README.md) 的架構圖與資料模型。

### 3.2 資料模型摘要
Collection `GuildData`，每個 Discord 伺服器一份文件，內含六個 Map：
`predictions`、`scores`、`match_history`、`match_schedules`、`match_teams`、`match_points`。
（欄位型別與語意詳見 README。）

### 3.3 啟動流程
```
檢查 MONGO_URI → mongoose.connect → Express.listen
  → client.login → client "ready" → 註冊全域斜線指令
```

## 5. 關鍵技術決策紀錄

| 決策 | 理由 | 取捨 |
|------|------|------|
| MongoDB Map of Mixed + 點號路徑 `$set` | 可對巢狀單一欄位做原子寫入，避免整份回寫 | 讀取時值為 plain object，需注意型別 |
| 分數一律 `$inc` | 消除讀-改-寫競態 | 無法在同一原子操作內讀到加分前的值 |
| 結算守衛放進 query + `matchedCount` | 防止並發重複結算（TOCTOU） | 邏輯較直覺寫法稍複雜 |
| 全域斜線指令 | 設定簡單、跨所有伺服器可用 | 指令定義變更需時生效（最多約 1 小時） |
| 單檔 `index.js` | 專案規模小，部署單純 | 檔案偏長，未來成長需模組化 |

## 6. 風險與緩解

| 風險 | 影響 | 緩解措施 |
|------|------|----------|
| 無自動化測試 | 改動易回歸 | 短期靠 `node --check` + 人工驗證；M6 導入測試 |
| MongoDB 連線中斷 | 指令全面失敗 | Mongoose 內建重連；可加連線狀態監控 |
| Discord API 速率限制 | `leaderboard` 等抓名稱變慢 | 已並發化；可加快取 |
| 環境變數外洩 | Bot 被盜用 | `.env` 已 gitignore；Token 僅存於 Zeabur Variables |
| 資料異常（舊資料缺欄位） | 計分迴圈 crash | 已加 `typeof` 與排程有效性守衛 + 日誌 |
